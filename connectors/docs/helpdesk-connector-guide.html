Developer guide

Help Desk Connector

A practical guide for integrating Android, Web, and .NET software with the Switch&Save internal Help Desk Bot. The connector sends reviewed software documentation and approved action definitions. Live business data stays inside the customer system.

How The Connector Works

1. Discover

Read routes, menus, screens, fields, validation rules, permissions, and service actions from the app.

2. Edit

Show a proper editable preview. Developers should not have to edit raw JSON as the normal workflow.

3. Audit

Check schema, handlers, navigation, confirmation rules, unsafe content, and missing documentation.

4. Sync

Send docs and action manifest as drafts. Admin approval is required before the bot can use docs.

Connector Studio

Connector Studio is the local preview/editor experience. It may be a Web UI, desktop screen, Android debug screen, or console UI for the first release.

Area What It Shows What Developer Can Do
Screens Module, screen, path, purpose, steps, fields, common errors. Edit, accept AI-generated draft, ignore, or mark needs review.
Actions Standard action library, connection status, risk, required fields, handlers. Connect local handler, disable, change roles, or mark custom.
Navigation Open-screen button labels, route IDs, Android routes, Web URLs, .NET commands. Test clickable paths and fix broken targets.
Changes New, updated, removed, and unchanged screens/actions since last saved manifest. Accept update, edit first, ignore, or remove from platform.
Audit Passed checks, warnings, and blocked issues. Fix blockers before sync.

Manifest Concept

The connector manifest has two main parts: documentation and actions.

{
  "appVersion": "1.0.0",
  "documents": [
    {
      "externalKey": "inventory.add_product",
      "module": "Inventory",
      "screen": "Add Product",
      "path": "Inventory > Products > Add Product",
      "purpose": "Create a new product.",
      "steps": ["Open Inventory", "Open Products", "Choose Add Product", "Save"],
      "fields": [
        { "name": "product_name", "required": true, "description": "Product display name" }
      ],
      "commonErrors": ["SKU already exists"],
      "actions": ["search_product", "create_product"]
    }
  ],
  "actions": [
    {
      "name": "search_product",
      "description": "Search products by name or SKU.",
      "type": "read",
      "risk": "low",
      "requiredFields": ["query"],
      "optionalFields": [],
      "allowedRoles": ["admin", "manager", "cashier"],
      "needsConfirmation": false
    }
  ]
}

Standard Action Library

Developers should connect these ready-made actions to their real services. Custom actions are allowed, but standard names make the bot easier to train and support.

Group Actions
Products search_product, get_product, create_product, update_product, update_product_price, update_product_quantity, disable_product
Stock check_stock, low_stock_products, stock_adjustment_history
Customers search_customer, create_customer, update_customer, update_customer_phone
Orders and Invoices search_order, get_order_status, create_order, cancel_order, search_invoice, get_invoice
Reports daily_sales_report, end_of_day_report, stock_value_report
Support create_support_ticket, add_customer_note

Clickable Paths

A path such as Inventory > Products > Add Product can become clickable only when the app exposes a route, URL, command, URI, or deep link. 

{
  "label": "Open Add Product",
  "routeId": "inventory.add_product",
  "platformTargets": {
    "android": { "route": "inventory/products/new", "deepLink": "mypos://inventory/products/new" },
    "web": { "url": "/inventory/products/new" },
    "dotnet": { "command": "OpenAddProductScreen", "uri": "mypos://inventory/products/new" }
  }
}

Audit Rules

  • Pass: all required fields exist, names are unique, handlers are connected.
  • Warn: short steps, missing navigation target, missing optional roles.
  • Block: secret detected, no handler for enabled action, dangerous action enabled, write action without confirmation.

The connector should save locally first, then audit, then sync. Switch&Save stores synced docs as drafts until a company admin approves them.

Platform Notes

Android

Use navigation routes or deep links. Store the token securely. Validate local staff role before write handlers.

Web

Run the connector token on the backend. Frontend admin UI can show Connector Studio and call backend sync/audit routes.

.NET

Prefer app commands/services over UI automation. External workers can use custom URI, named pipe, or localhost command bridge.
